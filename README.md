第二次随堂作业：电商商品列表⻚与详情⻚开发
1. 项目需求：实现电商平台的商品列表⻚（支持商品筛选、排序、分⻚）与商品详情⻚（展示商品信息、规格选择、加入购物⻋），要求组件化拆分合理、状态管理清晰、支持响应式
2. 实战步骤：
- 第一步：项目初始化（使用Vite/Rspack创建项目，配置UI库、状态管理库（如ReduxToolkit/Zustand）
- 第二步：组件拆分，将⻚面拆分为通用组件（导航栏、分⻚器、筛选组件）、业务组件（商品卡片、规格选择器、购物⻋弹窗）、⻚面组件（商品列表⻚、商品详情⻚）
- 第三步：实现组件通信与状态管理，通过Props实现父子组件数据传递，使用状态管理库存储全局数据（商品列表、筛选条件、购物⻋数据）
- 第四步：开发响应式UI，利用UI库与自定义样式，确保⻚面在电脑、平板、手机端均有良好展示效果
- 第五步：模拟数据交互（使用Mock.js生成商品数据），实现商品筛选（价格、分类）、排序（价格高低、销量）、分⻚加载、加入购物⻋功能
- 第六步：代码优化与调试，检查组件复用性、状态管理合理性，解决响应式适配问题与交互bug

- 项目功能概述
这是一个基于React的电商网站应用，实现了以下核心功能：

- 商品列表展示 ：支持价格筛选、排序和分页功能
- 商品详情页 ：展示商品详细信息，支持规格选择
- 购物车管理 ：添加商品到购物车、调整商品数量、删除商品、清空购物车
- 响应式设计 ：适配不同屏幕尺寸的设备
  技术栈
- 前端框架 ：React 19
- 路由管理 ：React Router 7.9.6
- UI组件库 ：Ant Design 6
- 状态管理 ：Zustand 5
- 构建工具 ：Vite 7.2.4
- 图标库 ：@ant-design/icons 6.1.0
设计实现思路
1. 项目结构
采用了清晰的模块化结构，将代码分为以下几个主要部分：

- pages/ ：包含主要页面组件（首页、商品列表、商品详情、购物车）
- components/ ：包含可复用的UI组件（ProductCard、FilterPanel、Pagination等）
- store/ ：使用Zustand实现的状态管理
- styles/ ：全局样式和响应式样式
2. 页面与路由设计
应用使用React Router实现路由管理，定义了4个主要路由：

- / ：首页
- /products ：商品列表页
- /products/:id ：商品详情页
- /cart ：购物车页
页面布局采用Ant Design的Layout组件，包含Navbar导航栏、Content内容区和Footer页脚。

3. 状态管理设计
使用Zustand实现轻量级状态管理，主要包含两个store：

- useProductStore：
  
  - 管理商品数据和筛选条件
  - 提供商品筛选、排序、分页等功能
  - 提供获取商品详情的方法
- useCartStore：
  
  - 管理购物车商品数据
  - 提供添加、删除、清空购物车等功能
  - 处理商品数量更新逻辑
4. 核心功能实现 商品列表功能
- 支持按价格范围筛选商品
- 支持按价格升降序和销量排序
- 实现分页展示，每页显示指定数量的商品
- 使用Grid布局展示商品卡片 商品详情功能
- 展示商品图片、名称、价格、销量等基本信息
- 支持商品规格选择
- 实现加入购物车功能，并提示商品是否已在购物车中 购物车功能
- 使用Ant Design的Table组件展示购物车商品
- 支持增加/减少商品数量
- 支持删除单个商品和清空购物车
- 实时计算购物车商品总价和总数量
### 5. 响应式设计
项目实现了响应式布局，通过CSS媒体查询和Ant Design组件的响应式属性，确保在不同屏幕尺寸下都能提供良好的用户体验。
